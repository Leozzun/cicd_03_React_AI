# React 전환 작업 계획서 (Vite + Styled Components)

본 프로젝트는 기존 퍼블리싱된 정적 웹 리소스(HTML/CSS/JS)를 React 기반의 현대적인 웹 애플리케이션으로 전환하는 것을 목표로 합니다.

## 1. 프로젝트 개요
- **대상**: 메인 페이지 (`index.html`) 및 관련 자산
- **프레임워크**: React (Vite)
- **언어**: JavaScript
- **스타일 라이브러리**: Styled Components
- **주요 기능**: 메인 슬라이더 (Swiper), 네비게이션 GNB, 반응형 레이아웃

## 2. 작업 환경 구성
### 2.1 의존성 설치
- `vite`: 프로젝트 빌드 도구
- `react`, `react-dom`: 핵심 라이브러리
- `react-router-dom`: 라우팅 관리
- `styled-components`: 스타일 관리
- `swiper`: 슬라이더 기능 구현
- `lucide-react`: 아이콘 (필요 시)

### 2.2 디렉토리 구조 (`src/`)
```text
src/
├── assets/             # 이미지, 폰트 등 정적 자산
├── components/         # 공통 컴포넌트 (Header, Footer, Layout 등)
├── pages/              # 페이지 컴포넌트 (Home 등)
├── styles/             # GlobalStyle 및 Theme 정의
└── App.jsx             # 라우팅 및 전역 설정
```

## 3. 세부 작업 단계

### 단계 1: 프로젝트 초기화 및 자산 이동
- `npm create vite@latest`를 통한 프로젝트 생성
- 기존 `img/` 폴더의 리소스를 `public/img/` 또는 `src/assets/img/`로 이동
- 기존 폰트 및 웹 폰트 설정 (Raleway 등)

### 단계 2: 전역 스타일 설정 (Styled Components)
- `src/styles/GlobalStyle.js` 생성: 기존 `reset.css`, `common.css` 내용을 기반으로 기본 스타일 정의
- 테마 설정 (Color Palette, Breakpoints 등)

### 단계 3: 공통 레이아웃 분리
- **Header (`src/components/Header.jsx`)**: 
  - GNB(대메뉴/소메뉴) 및 전체메뉴(All Menu) 로직 구현
  - 기존 jQuery 기반의 메뉴 토글 로직을 React `useState`로 전환
- **Footer (`src/components/Footer.jsx`)**: 회사 정보 및 로고 영역 분리
- **Layout (`src/components/Layout.jsx`)**: Header와 Footer를 포함하는 공통 레이아웃 컴포넌트

### 단계 4: 페이지 컴포넌트 개발
- **Home (`src/pages/Home.jsx`)**: `index.html`의 메인 컨텐츠 영역 구현
  - **Visual Section**: `swiper/react`를 사용하여 메인 슬라이더 구현
  - **Product Section**: 제품 목록 루프 및 슬라이더 구현
  - **Construction Case**: 시공 사례 리스트 구현
  - **BBS Wrap**: 공지사항, 고객센터, 견적문의 영역 구현

### 단계 5: 스타일 마이그레이션
- 기존 `content.css` 및 기타 CSS 파일의 스타일을 각 컴포넌트별 `styled-components`로 분리 및 이관
- 클래스 기반 선택자를 변수화하여 관리

### 단계 6: 인터랙션 및 라우팅
- `react-router-dom`을 사용한 기본 경로(`/`) 설정
- jQuery 의존성 제거 및 순수 React/JS로 인터랙션(애니메이션, 클릭 이벤트 등) 재구현

## 4. 고려 사항
- **이미지 경로**: React 빌드 환경에 맞는 경로 설정 확인 (`process.env.PUBLIC_URL` 또는 상대 경로)
- **Swiper 버전**: 기존 Swiper 4.2에서 최신 `swiper/react` 라이브러리로 업그레이드 및 옵션 조정
- **반응형 대응**: 기존 미디어 쿼리 내용을 Styled Components 내에 포함하여 유지 관리성 향상
